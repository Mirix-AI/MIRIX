Three-Tier Architecture

1
MirixClient mirix/client/remote_client.py
Async client-side HTTP wrapper implementing AbstractClient interface
↓ HTTP/HTTPS
2
REST API (FastAPI) mirix/server/rest_api.py
Async HTTP endpoints that route requests to AsyncServer
↓ Method Calls
3
AsyncServer mirix/server/server.py
Async core business logic, manages agents, tools, memory, database
Key Point: The REST API (rest_api.py) is a thin async wrapper around AsyncServer (server.py). MirixClient makes HTTP requests to REST API endpoints, which then call corresponding AsyncServer methods. The entire stack is async-native — all I/O uses asyncpg, redis.asyncio, aiokafka, and httpx.AsyncClient. 
# Example: How a request flows through the system

# 1. Client makes a call
client = MirixClient(base_url="http://localhost:8531")
agent = await client.create_agent(name="test")

# 2. MirixClient._request() makes HTTP POST to /agents
POST http://localhost:8531/agents
Headers: { "X-API-Key": "..." }
Body: { "name": "test", ... }

# 3. FastAPI endpoint receives and routes
@app.post("/agents")
async def create_agent(request: CreateAgentRequest):
    server = get_server()  # Get AsyncServer singleton
    return await server.create_agent(...)

# 4. AsyncServer processes the request
class AsyncServer:
    async def create_agent(self, request, actor):
        agent_state = await self.agent_manager.create_agent(...)
        return agent_state

Context Window Summarization

Why Summarize?

LLMs have a fixed context window (e.g., 128K tokens). When the conversation history exceeds a threshold (default: 75% of context window), older messages are summarized to make room for new ones. Target after summarization: 10% memory pressure. Last 5 messages are always preserved.

Heartbeats & Chaining

What is Chaining?

Chaining allows an agent to take multiple actions before responding. The agent loops until it decides to stop.
Why? Complex tasks require multiple steps: search memory, analyze, then respond.

What are Heartbeats?

Heartbeats are synthetic messages injected between chaining steps to keep the LLM conversation flowing. 
# mirix/constants.py
REQ_HEARTBEAT_MESSAGE = "[System] Function called using continue_chaining=true, returning control"
FUNC_FAILED_HEARTBEAT_MESSAGE = "[System] Function call failed, returning control"

Chaining Loop Logic

while True:
    step_response = await self.inner_step(next_input_message)

    if function_name == "send_message":  # chat_agent done
        continue_chaining = False
    elif function_name == "finish_memory_update":  # memory agent done
        continue_chaining = False

    if not continue_chaining:
        break
    elif function_failed:
        next_input = FUNC_FAILED_HEARTBEAT_MESSAGE
    else:
        next_input = REQ_HEARTBEAT_MESSAGE
